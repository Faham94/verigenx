## 2. **CRITICAL CAVEAT: Functional Coverage Proxy Approximations**

> [!WARNING]
> **Measurement Disclaimers**:
> The SystemVerilog testbench templates (`coverage.sv.j2`) utilize a mock subscriber model with **manual hit-counters** as proxy indicators for functional coverage rather than native SystemVerilog covergroups/coverpoints. 
> 
> These conditions (e.g. `if (t.rx_data != 0) hit_FP_001++`) are basic approximations based on transaction values rather than complete protocol-aware bin coverage:
> - **UART**: Data transmission is approximated by non-zero transaction variables.
> - **SPI / I2C**: Reset and select assertions are checked via simple port values.
> 
> The CoverHunter feedback loop successfully identified these uncovered conditions and generated directed test sequences (such as `uart_test_directed_IDLE.sv`) that drove these specific proxy conditions to hit-count completion, but this should be recognized as a mock-based approximation.

## 3. Targeted Directed Test Generation Example

Here is an example of a targeted directed test generated by `CoverHunter` during verification to cover the `IDLE` state / data transfer gaps:

```systemverilog
class uart_sequence_IDLE extends uvm_sequence #(uart_seq_item);
    `uvm_object_utils(uart_sequence_IDLE)
    function new(string name = "uart_sequence_IDLE");
        super.new(name);
    endfunction
    virtual task body();
        req = uart_seq_item::type_id::create("req");
        start_item(req);
        if (!req.randomize()) begin
            `uvm_fatal("SEQ", "Randomization failed")
        end
        req.tx_data = 8'hAA;
        req.rx_data = 8'hAA;
        finish_item(req);
    endtask
endclass

class uart_test_directed_IDLE extends uart_test_base;
    `uvm_component_utils(uart_test_directed_IDLE)
    function new(string name = "uart_test_directed_IDLE", uvm_component parent = null);
        super.new(name, parent);
    endfunction
    virtual task run_phase(uvm_phase phase);
        uart_sequence_IDLE seq;
        phase.raise_objection(this);
        seq = uart_sequence_IDLE::type_id::create("seq");
        seq.start(env.agent.sequencer);
        phase.drop_objection(this);
    endtask
endclass
```

## 4. Loop Stagnation & Rollback Verification

- **Convergence detection**: The loop correctly terminates if target functional coverage reaches 85%+ or if the functional coverage improvement delta between iterations drops below the convergence threshold (`0.1%`).
- **Rollback mechanism**: If a newly generated test fails compilation (e.g. invalid syntax) or reduces overall coverage metrics (VCD compilation/verification failure), the feedback loop successfully:
  1. Detects the decline/error.
  2. Discards the new test file from the UVM files list.
  3. Deletes the physical file from `generated_uvm/`.
  4. Restores the last known-good test suite status.
